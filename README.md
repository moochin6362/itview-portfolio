## 💡 2. 개발 배경 및 목적

### ▪ K-뷰티 시장 성장
- 글로벌 K-뷰티 시장은 2023년 약 **141억 달러 → 2032년 280억 달러** 규모로 성장 전망  
- 화장품 관련 정보는 많지만, 자신의 **피부 상태에 맞는 제품을 효율적으로 찾기 어려움**

### ▪ 개인 맞춤형 서비스 수요 증가
- 사용자는 **성분, 리뷰, 피부 고민** 기반으로 정교한 추천을 원함  
- MZ세대의 **72% 이상이 인플루언서 리뷰 기반 구매 경험**이 있음  
- 이에 따라 **리뷰 기반 체험단 시스템 + 추천 알고리즘**을 결합한 서비스 개발

## 🧱 6. 주요 기능

### 👤 마이페이지
- 회원 정보 조회/수정 (프로필 이미지, 피부 정보 등)
- 포인트 적립·소멸 내역, 쿠폰 목록 조회
- 내가 신청한 체험단 / 작성 리뷰 관리

### 🧴 체험단 신청
- 모집글별 상세 보기 및 신청 폼 연결
- 로그인한 사용자의 `userNo` 기준으로 신청 이력 저장
- 모집 마감 시점에 따라 버튼 비활성화 처리  
  (→ JavaScript에서 `disabled` 제어)

### ⭐ 리뷰 작성
- Toast UI Editor를 활용한 텍스트+이미지 리뷰 작성
- 첨부 이미지 자동 저장 및 썸네일 미리보기
- 비회원 접근 시 로그인 유도 처리

### 💰 포인트·쿠폰
- 상품 구매 및 리뷰 작성 시 포인트 자동 적립
- 스케줄러 기반 포인트 만료 처리
- 쿠폰은 등급별/이벤트별 발급 및 기간 만료 관리

### ① 모집글 번호 받아오기 (URL → hidden input 세팅)

![체험단신청0](./img/체험단신청0.png)

#### 💡 문제 상황  
신청 버튼을 눌렀을 때 어떤 모집글을 선택했는지 서버로 전달되지 않음 → `expNo` 값이 null.

#### 🧩 원인  
URL에는 `?expNo=123`이 붙어 있지만, 폼 내부에 해당 값을 세팅하는 코드가 없음.

#### ✅ 해결 방법  
`URLSearchParams`로 URL의 expNo를 읽어와 숨겨진 input 태그(`id="expNo"`)에 자동 세팅.
